Software Factory · Backlog ordering

Rank epics and projects by priority, ignoring done stories

ALF-49 Fix epic ordering so completed work no longer drags a finished-up epic to the top, and extend the same priority ranking to the project list in the sidebar.

Problem

The Code module orders work by each story's global Backlog priority (lower = higher priority). The per-project board already ordered epics by their best story's priority, but it counted every story — including done and abandoned ones. A single high-priority completed story therefore kept an epic whose remaining open work is low-priority pinned to the top of the board, which is misleading: ranking should reflect outstanding work, the same set the Backlog shows by default.

Separately, the project list in the sidebar was rendered in seed order with no notion of priority, so the sidebar didn't lead with the project that holds the most urgent open work.

Change

1. Exclude done/abandoned stories from epic rank

An epic's rank is the best (lowest) priority across its outstanding stories only — everything except done and abandoned, mirroring the Backlog's own isBacklogOutstanding filter. An epic with no outstanding story (none, or all done/abandoned) has no rank and sorts last, with created_at ascending as the stable tie-break. This is a one-line filter added inside epicRank in lib/stores/code-store.tsx; within-lane and escape-bucket ordering are unchanged.

2. Rank the sidebar project list by priority

A new derived hook useRankedProjects orders projects by each project's best outstanding story's priority — the same rule the board applies to epics, one level up — with created_at ascending as the tie-break for projects with no outstanding work. ProjectNav consumes the ranked list instead of the raw useProjects. The raw useProjects accessor keeps returning store order for other callers.

Out of scope

  • No schema or API change — ranking is purely client-side over the existing priority.
  • No change to the Backlog list, within-lane order, or the reorder/move RPCs.
  • Archived epics still split out of the active board and rank among themselves.

Acceptance

  • An epic whose only high-priority story is done ranks by its outstanding stories; an all-done epic sorts last.
  • The sidebar lists projects ordered by their best outstanding story's priority.
  • Done/abandoned stories never influence either the epic rank or the project rank.

Demonstrated in docs/demos/ALF-49/epic-and-project-ranking.md and covered by unit tests in lib/stores/code-store.test.tsx and components/code/project-nav.test.tsx.
